# spec.md — URL to Markdown

## 개요
사용자가 웹 페이지 URL을 붙여넣으면 본문을 Markdown으로 추출해 화면에 보여주고, 복사·다운로드하거나 ChatGPT·Claude로 바로 이어서 열 수 있게 하는 서비스.

## 범위

### 포함
- URL 입력 → 본문을 Markdown으로 변환해 제목·저자 헤더와 함께 표시
- 변환 결과 복사하기 / .md 파일 다운로드
- 변환 결과를 ChatGPT 또는 Claude에서 이어서 열기 (프롬프트 선택적으로 앞에 붙이기)
- 프리셋 프롬프트(요약해줘 / 한국어로 번역해줘 / 쉽게 설명해줘) 선택
- 프리셋 대신 직접 프롬프트 입력 (저장하지 않는 1회성 입력)
- 다크모드 토글 (기본값은 시스템 설정 따름)
- 입력·결과 초기화(지우기)

### 제외
- 로그인/계정, 변환 이력 저장 — 게스트 전용, 매 방문 무상태 서비스로 정의됨 (사용자 확인 완료)
- 여러 URL 일괄 변환 — 이번 범위는 URL 1개 처리로 한정 (사용자 확인 완료)
- ChatGPT·Claude 공식 프리필 API 연동 — 두 서비스 모두 URL로 텍스트를 미리 채우는 공식 기능이 없어, 클립보드 복사 + 새 탭 열기 방식으로 대체 (사용자 확인 완료)

## 시나리오

### 1. URL 입력 후 변환 성공
- **Given** — 입력 화면이 비어 있는 상태
- **When** — URL 입력칸에 유효한 페이지 URL을 붙여넣고 "변환" 버튼을 클릭
- **Then** — 결과 화면으로 전환되며 페이지 제목 헤더와 렌더링된 Markdown 본문이 표시된다

성공 기준:
- [ ] 유효한 기사 URL 붙여넣기 → "변환" 클릭 → 제목이 헤더에 표시되고 본문이 Markdown으로 렌더링된 상태로 화면 전환
- [ ] 저자 정보가 있는 페이지 → 결과 헤더에 저자명이 표시됨

### 2. 저자 정보 없는 페이지 변환
- **Given** — 저자 메타데이터가 없는 페이지의 URL
- **When** — "변환" 버튼 클릭으로 변환 완료
- **Then** — 결과 헤더에 저자 항목 자체가 나타나지 않는다 (제목만 표시)

성공 기준:
- [ ] 저자 정보 없는 URL 변환 → 결과 헤더에 저자 필드가 렌더링되지 않음

### 3. URL 변환 실패
- **Given** — 입력 화면이 비어 있는 상태
- **When** — 잘못된 형식의 URL, 접근 불가능한 페이지, 또는 본문 추출이 불가능한 페이지의 URL로 "변환" 버튼을 클릭
- **Then** — 원인 구분 없이 동일한 공통 에러 메시지가 표시되고 결과 화면으로 전환되지 않는다

성공 기준:
- [ ] 형식이 잘못된 문자열 입력 → "변환" 클릭 → 공통 에러 메시지 표시, 결과 화면 미전환
- [ ] 접근 불가능한 URL(404 등) → "변환" 클릭 → 동일한 공통 에러 메시지 표시
- [ ] 본문 추출 실패 페이지 → "변환" 클릭 → 동일한 공통 에러 메시지 표시

### 4. 지우기로 초기화
- **Given** — URL 변환 결과가 표시된 상태
- **When** — "지우기" 버튼 클릭
- **Then** — URL 입력칸과 변환 결과가 모두 비워지고 최초 입력 화면으로 돌아간다

성공 기준:
- [ ] 결과 표시 상태에서 "지우기" 클릭 → 입력칸 빈 값, 결과 영역 미표시(최초 화면)

### 5. Markdown 복사하기
- **Given** — 변환 결과가 표시된 상태 (프롬프트 선택 여부 무관)
- **When** — "복사하기" 클릭
- **Then** — 프롬프트가 포함되지 않은 순수 Markdown 텍스트가 클립보드에 복사되고, 복사됐다는 피드백이 표시된다

성공 기준:
- [ ] 프리셋 프롬프트가 선택된 상태에서 "복사하기" 클릭 → 클립보드 내용은 순수 Markdown만 포함 (프롬프트 미포함), 복사 완료 피드백 표시

### 6. .md 파일 다운로드
- **Given** — 변환 결과가 표시된 상태
- **When** — ".md 파일 다운로드" 클릭
- **Then** — 순수 Markdown 내용을 담은 .md 파일이 다운로드된다

성공 기준:
- [ ] 변환 결과 상태에서 다운로드 클릭 → .md 확장자 파일이 다운로드되고 내용은 순수 Markdown과 일치

### 7. 프롬프트 없이 ChatGPT·Claude로 열기
- **Given** — 변환 결과가 표시된 상태, 프롬프트 프리셋 미선택
- **When** — "ChatGPT로 열기"(또는 "Claude로 열기") 클릭
- **Then** — Markdown 본문이 클립보드에 복사되고 chatgpt.com(또는 claude.ai)이 새 탭으로 열리며, "클립보드에 복사됨, 붙여넣으세요" 안내가 표시된다

성공 기준:
- [ ] 프롬프트 미선택 상태에서 "ChatGPT로 열기" 클릭 → 새 탭에서 chatgpt.com 열림, 클립보드는 Markdown 본문만 포함, 안내 토스트 표시
- [ ] 동일 조건에서 "Claude로 열기" 클릭 → 새 탭에서 claude.ai 열림, 클립보드는 Markdown 본문만 포함

### 8. 프리셋 프롬프트와 함께 열기
- **Given** — 변환 결과가 표시된 상태
- **When** — 프리셋 프롬프트(예: "요약해줘") 선택 후 "ChatGPT로 열기" 클릭
- **Then** — 선택한 프롬프트 문구가 앞에 붙은 텍스트가 클립보드에 복사되고 chatgpt.com이 새 탭으로 열린다

성공 기준:
- [ ] "요약해줘" 프리셋 선택 후 "ChatGPT로 열기" 클릭 → 클립보드 내용이 "요약해줘" 문구로 시작하고 이어서 Markdown 본문이 포함됨

### 9. 직접 입력 프롬프트와 함께 열기
- **Given** — 변환 결과가 표시된 상태, 프리셋 옆 "직접 입력" 선택으로 텍스트 인풋이 노출된 상태
- **When** — 직접 입력 필드에 임의의 프롬프트 문구를 입력하고 "ChatGPT로 열기" 클릭
- **Then** — 입력한 문구가 앞에 붙은 텍스트가 클립보드에 복사되고 chatgpt.com이 새 탭으로 열린다. 입력한 문구는 어디에도 저장되지 않는다

성공 기준:
- [ ] 직접 입력 필드에 임의 문구 입력 → "ChatGPT로 열기" 클릭 → 클립보드가 해당 문구로 시작
- [ ] 새로고침 또는 새 변환 이후 직접 입력 필드는 항상 빈 상태로 시작 (이전 입력이 남아있지 않음)

### 10. 프리셋 vs 직접 입력 전환
- **Given** — "직접 입력" 선택으로 텍스트 인풋이 노출되고 문구가 입력된 상태
- **When** — 프리셋 옵션(예: "요약해줘")을 다시 선택
- **Then** — 직접 입력 텍스트 인풋이 사라지고 선택한 프리셋 문구가 적용된다

성공 기준:
- [ ] 직접 입력 상태에서 프리셋 선택 → 직접 입력 인풋 숨김, 프리셋 문구가 이후 "열기" 동작에 적용됨

### 11. 다크모드 토글
- **Given** — 시스템이 다크 모드로 설정된 환경에서 처음 페이지 접속
- **When** — 페이지가 로드됨
- **Then** — 다크모드가 기본 적용된 화면이 표시된다

성공 기준:
- [ ] 시스템 다크모드 환경에서 첫 접속 → 다크 테마로 렌더링
- [ ] 다크모드 토글 클릭 → 라이트/다크 테마가 즉시 전환됨

## 불변 규칙

- 프롬프트(프리셋이든 직접 입력이든)는 "복사하기"와 ".md 다운로드" 결과물에는 절대 포함되지 않는다 — 오직 "ChatGPT·Claude로 열기" 동작에서 클립보드에 앞에 붙는 형태로만 반영된다
- 직접 입력한 프롬프트 문구는 어떤 저장소에도 영속 저장되지 않는다 (세션 내 1회성)
- URL 변환이 실패하는 모든 원인(잘못된 형식, 접근 불가, 추출 실패)은 사용자에게 동일한 단일 공통 에러 메시지로 노출된다 — 실패 사유를 구분해 보여주지 않는다
